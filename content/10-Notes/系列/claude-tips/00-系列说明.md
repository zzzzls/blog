囤一些用 Claude / Claude Code 的小技巧——hook、slash command、MCP、prompt 模板、工作流取舍之类。每条都标注了**在哪个 Claude 版本上验证过**，因为这类工具迭代很快，半年前管用的写法今天可能已经被官方改名/淘汰。

## 这个系列的 frontmatter 约定

每篇笔记除了通用字段之外，多两个：

- `claude_version`：这条 tip 最后一次被验证可用的 Claude 版本号（如 `2.1.133`）。读者据此判断「我现在用的版本是不是还能照搬」。
- `category`：tip 的子分类。当前在用的取值：

| category | 含义 |
|---|---|
| `hooks` | Claude Code 的 hook 系统（PreToolUse / PostToolUse 等） |
| `slash-commands` | 自定义 / 内置 slash 命令 |
| `mcp` | MCP server 集成、配置、自建 |
| `prompts` | prompt 工程、CLAUDE.md / AGENTS.md 写法 |
| `workflow` | 工作流模式（plan mode、multi-agent、subagent 调度等） |
| `meta` | 系列自身的说明、目录页等 |

新加分类前先想想能不能并入现有的——目录页爆 10 个就该重新整理了。

## 为什么把版本写在 frontmatter 里

Claude / Claude Code 的迭代很激进。一条「今天好用」的小技巧，明年这个时候大概率：API 改了、配置字段名变了、或者被新的内置功能替代。把版本明确标出来，读者扫一眼就知道这条 tip 的「保鲜期」到哪儿。

博客上每篇笔记的标题旁会渲染出版本号 + 分类（由 `zzzzls/blog` 的自定义 Quartz 组件 `ClaudeMeta` 实现），不用读者自己翻 frontmatter。
